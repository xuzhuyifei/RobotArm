# 机械臂控制系统 - Main GUI

## 简介

`main.py` 是机械臂控制系统的主界面程序，提供图形化界面（GUI）用于控制机械臂的运动、末端执行器和程序管理。

## 功能特性

### 1. 串口连接管理
- 自动检测可用串口
- 支持波特率设置（9600/115200）
- 实时连接状态显示

### 2. 运动控制

#### 关节控制
- **关节运动**：直接设置关节角度（A1, A2, A3）
- **关节偏移**：相对当前位置偏移关节角度

#### 世界坐标控制
- **走点**：直接设置世界坐标位置（X, Y, Z）
- **偏移**：相对当前位置偏移世界坐标

#### 直线运动
- **直线**：直线路径移动到目标位置
- **直线偏移**：相对当前位置直线偏移

### 3. 末端执行器控制
- **吸嘴控制**：开关吸嘴（0/1）
- **抓手控制**：开关抓手（0/1）

### 4. 系统控制
- **回原点**：机械臂回到零点位置
- **急停**：紧急停止所有运动
- **查询参数**：获取机械臂当前状态信息

### 5. 步骤缓冲区管理

#### 录制功能
- 点击"开始录制步骤"后，所有运动指令会自动写入步骤缓冲区
- 支持设置默认参数：延时、IO 引脚、IO 状态

#### 插入模式
- **智能插入**：选中某个步骤后，新指令将插入到该步骤之后
- **替换空步骤**：选中空步骤时，直接替换为空步骤内容
- **插入空步骤**：可在任意位置插入空步骤占位

#### 步骤操作
- 删除指定步骤
- 清空本地显示（不影响下位机）

### 6. 程序配方管理

#### 配方操作
- **写入配方 (SetStepOK)**：将当前步骤缓冲区写入指定配方号
- **清除配方 (Clean)**：清空指定配方号的程序
- **编辑配方 (Editor)**：进入配方编辑模式

#### 运行控制
- **单循环 (Single)**：单次运行指定配方
- **循环运行 (Cycle)**：持续循环运行指定配方
- **查询配方 (Refer)**：查看当前配方内容

#### 高级设置
- **自动校准 (Align)**：设置自动校准次数
- **DI 触发 (Trigger)**：启用外部 DI 信号触发循环运行

## 使用方法

### 启动程序
```bash
python main.py
```

### 基本操作流程

1. **连接串口**
   - 选择正确的串口号
   - 选择波特率（默认 115200）
   - 点击"连接"按钮

2. **手动控制测试**
   - 在"运动"标签页测试关节和世界坐标运动
   - 在"末端"标签页测试吸嘴和抓手
   - 在"系统"标签页执行回原点、急停等操作

3. **录制程序**
   - 打开"步骤缓冲区"面板
   - 点击"开始录制步骤"
   - 执行所需的运动和控制指令
   - 点击"停止录制步骤"

4. **保存配方**
   - 选择配方号（1/2/3）
   - 点击"写入配方 (SetStepOK)"

5. **运行程序**
   - 选择要运行的配方号
   - 点击"单循环"或"循环运行"

## 注意事项

1. **串口连接**
   - 必须先连接串口才能执行控制指令
   - 连接状态下其他控制按钮才会激活

2. **步骤录制**
   - 录制模式下，每个运动指令都会发送到下位机并写入缓冲区
   - 可以通过选中步骤来指定插入位置

3. **安全提示**
   - 首次运行建议在虚拟串口环境测试
   - 确保机械臂工作空间内无障碍物
   - 急停按钮可随时停止所有运动

## 测试环境

### 虚拟串口测试（推荐新手）
1. 使用 com0com 创建虚拟串口对（如 COM10 <-> COM11）
2. 运行 `virtual_robot.py` 连接到 COM11
3. 运行 `main.py` 连接到 COM10
4. 可以安全测试所有功能而无需真实硬件

### 真实硬件测试
1. 用 USB 线连接机械臂控制器
2. 运行 `main.py` 选择对应的串口号
3. 连接后即可控制

## 常见问题

**Q: 点击连接没有反应？**
A: 检查是否选择了正确的串口号，确认设备已连接

**Q: 发送指令失败？**
A: 检查串口连接状态，确认波特率匹配

**Q: 步骤无法保存？**
A: 确保已连接串口，录制功能需要在连接状态下使用
